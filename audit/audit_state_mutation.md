# audit_state_mutation.md — state-mutation opcode audit

Scope: the 17 state-mutation opcodes named in the audit charter. For each
I checked which value-list bits get READ, which drive a real state change
or rendering effect (ACT-ON), which are STORED-BUT-INERT (persist across
round-trip via GetWindowAttributes / GetGCAttributes — passes tests,
affects nothing), and which are SILENTLY DROPPED entirely (never even
decoded).

## The pattern I noticed across this category

The CW/GC value-list family has a consistent two-layer shortcut: at the
**decode/store layer**, almost every bit is dutifully parsed and stashed
into `WindowEntry` or `GCEntry.values`, so round-trip reads (especially
GetWindowAttributes after the 2026-05-15 sweep) come back correct. But
at the **materialise / consume layer** — `GCState.materialise`, the
bridge's drawing methods, the render-time paint paths — only a fixed
hand-picked subset (the ones the test corpus actually drives) is
consulted. Everything else is **stored-but-inert**: tests pass, the wire
trace looks honest, and a real client outside our corpus that depends
on the documented effect gets nothing. The two highest-leverage cases
are **`GCBits.graphicsExposures`** (we always emit GraphicsExposure even
when the GC asked for none) and **`CW.bitGravity`** (we always full-
repaint on resize, regardless of what the client requested) — both
silently observable through the wire log as "we did the work the client
asked us NOT to" rather than the more obvious "we didn't do the work."

A second meta-pattern: substructure-redirect (`MapRequest`,
`ConfigureRequest`) is entirely absent. Anywhere the spec says "if
parent has SubstructureRedirectMask, redirect to the manager," we apply
the request unconditionally. This is invisible until the day we run
under a WM, then every map/configure desyncs.

Third: the ChangeProperty Prepend/Append type+format **mismatch
silent-overwrite** is the single most pernicious finding here — it's
the only one that corrupts data on the server side rather than just
ignoring a request, and the bug is invisible to the client until it
GetProperty's back the property and finds garbled bytes interleaved.
